============================================================
НАЗВАНИЕ ДИАГРАММЫ: Диаграмма архитектуры приложения
    (Architecture Diagram)
============================================================

ОПИСАНИЕ:
Диаграмма архитектуры отображает трёхуровневую структуру
веб-приложения «La Bella Italia» и потоки данных между
компонентами системы.

УРОВНИ АРХИТЕКТУРЫ:

1. КЛИЕНТСКИЙ УРОВЕНЬ (Frontend — Browser)
   Технологии: HTML5, CSS3, Vanilla JavaScript
   Компоненты:
   - HTML-страницы (index, menu, cart, auth, profile, admin, 404)
   - Единый файл стилей style.css
   - Модули JavaScript:
     * app.js — рендеринг страниц и бизнес-логика
     * auth.js — аутентификация на клиенте
     * cart.js — корзина (localStorage)
     * db.js — API-клиент (fetch-обёртка)
   - localStorage для хранения корзины и данных сессии

2. СЕРВЕРНЫЙ УРОВЕНЬ (Backend — Node.js)
   Технологии: Node.js, Express.js, Sequelize ORM
   Компоненты:
   - Express.js — маршрутизация HTTP-запросов
   - Middleware слой:
     * helmet — безопасность HTTP-заголовков
     * express-session — серверные сессии
     * express-rate-limit — защита от брутфорса
     * bcryptjs — хеширование паролей
   - NeDB-совместимая абстракция (database.js):
     транслирует запросы NeDB-стиля в Sequelize
   - Sequelize ORM (модели: User, MenuItem, Order)

3. УРОВЕНЬ ДАННЫХ (Database — PostgreSQL)
   Технологии: PostgreSQL 16, JSONB
   Компоненты:
   - Таблица users (пользователи)
   - Таблица menu_items (меню)
   - Таблица orders (заказы)

ПОТОК ДАННЫХ:
  Браузер → HTTP/JSON → Express.js → Middleware →
  → NeDB-обёртка → Sequelize ORM → PostgreSQL

ОСОБЕННОСТИ АРХИТЕКТУРЫ:
- NeDB-совместимая обёртка позволяет переключаться
  между PostgreSQL и встроенной NeDB без изменения кода
- JSONB-поля дают гибкость документной модели
  внутри реляционной СУБД
- Клиентская корзина хранится в localStorage —
  не нагружает сервер до момента оформления заказа
