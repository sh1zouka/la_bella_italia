============================================================
НАЗВАНИЕ ДИАГРАММЫ: Диаграмма классов
    (Class Diagram)
============================================================

ОПИСАНИЕ:
Диаграмма классов отображает структуру объектной модели
приложения «La Bella Italia»: три основных класса,
их атрибуты, методы и взаимосвязи.

КЛАССЫ СИСТЕМЫ:

1. КЛАСС User (Пользователь)
   Назначение: хранение данных пользователей и аутентификация
   Атрибуты:
   - id (PK) — уникальный идентификатор
   - name — имя пользователя
   - email — электронная почта (логин, UNIQUE)
   - password — хеш пароля (bcrypt, 10 раундов)
   - phone — номер телефона
   - role — роль: 'user' (клиент) или 'admin' (администратор)
   - bonusPoints — бонусные баллы (100 при регистрации)
   - createdAt — дата регистрации
   Методы:
   - register() — регистрация нового пользователя
   - login() — аутентификация (проверка пароля)
   - getProfile() — получение профиля
   - updateBonuses() — обновление бонусов после заказа
   - getOrders() — получение заказов пользователя
   Связи: 1 → * Order (один пользователь имеет много заказов)

2. КЛАСС MenuItem (Блюдо меню)
   Назначение: хранение информации о блюдах
   Атрибуты:
   - id (PK) — id блюда (1..50)
   - category — категория (pizza, pasta, starters, desserts, drinks)
   - name — название блюда
   - desc — описание и состав
   - price — цена в рублях
   - weight — вес/объём порции
   - img — emoji-иконка
   - photo — URL фотографии
   - macros (JSONB) — КБЖУ: {kcal, protein, fat, carbs}
   - popular — флаг «Хит продаж»
   Методы:
   - findAll() — получение всех блюд
   - findByCategory() — фильтрация по категории
   - search() — поиск по названию/описанию
   - create() — добавление нового блюда (админ)
   - deleteById() — удаление блюда (админ)

3. КЛАСС Order (Заказ)
   Назначение: хранение и обработка заказов
   Атрибуты:
   - id (PK) — номер заказа
   - userId (FK → User.id) — ID пользователя
   - items (JSONB) — состав заказа [{name, price, qty, img}]
   - total — сумма до скидки
   - bonusDiscount — скидка бонусами
   - finalTotal — итоговая сумма
   - earnedBonuses — начислено бонусов (5% от finalTotal)
   - deliveryData (JSONB) — данные доставки {name, phone, address}
   - status — статус: confirmed → preparing → delivering → done
   - createdAt — дата заказа
   Методы:
   - create() — создание заказа с расчётом бонусов
   - findByUser() — история заказов пользователя
   - findAll() — все заказы (админ)
   - updateStatus() — изменение статуса (админ)
   - deleteById() — удаление заказа (админ)
   - getStats() — получение статистики (админ)
   - calculateTotal() — расчёт суммы
   - calculateBonuses() — расчёт бонусов
   Связи: * → 1 User (многие заказы принадлежат одному пользователю)

СВЯЗИ МЕЖДУ КЛАССАМИ:
  User (1) ──── (N) Order

  Примечание: MenuItem не имеет прямой ассоциации с Order,
  так как состав заказа хранится в JSONB-поле items
  (документная модель внутри реляционной БД)

МЕТОДЫ АДМИНИСТРИРОВАНИЯ:
  Order.findAll() — просмотр всех заказов
  Order.updateStatus() — смена статуса
  Order.deleteById() — удаление заказа
  Order.getStats() — статистика (выручка, средний чек)
  MenuItem.create() — добавление блюда
  MenuItem.deleteById() — удаление блюда
