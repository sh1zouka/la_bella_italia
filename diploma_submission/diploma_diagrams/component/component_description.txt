============================================================
НАЗВАНИЕ ДИАГРАММЫ: Диаграмма компонентов
    (Component Diagram)
============================================================

ОПИСАНИЕ:
Диаграмма компонентов отображает структурную организацию
программных компонентов системы доставки «La Bella Italia»
и зависимости между ними. Показана декомпозиция системы
на клиентскую и серверную части.

КЛИЕНТСКИЕ КОМПОНЕНТЫ (Frontend):

1. HTML-СТРАНИЦЫ (7 страниц):
   - index.html — главная (hero, хиты, бонусы)
   - menu.html — меню с категориями и поиском
   - cart.html — корзина и форма заказа
   - auth.html — вход и регистрация
   - profile.html — профиль и история заказов
   - admin.html — панель администратора
   - 404.html — страница ошибки

2. CSS: style.css (единый файл стилей)

3. JAVASCRIPT МОДУЛИ (4 модуля):
   - db.js — API-клиент (fetch-обёртка для REST API)
   - auth.js — аутентификация (localStorage сессии)
   - cart.js — корзина (localStorage)
   - app.js — главная логика (рендеринг, валидация)

4. ХРАНИЛИЩЕ: localStorage (корзина и сессия)

СЕРВЕРНЫЕ КОМПОНЕНТЫ (Backend):

1. ЯДРО СЕРВЕРА:
   - server.js — Express.js с 14 API-эндпоинтами
   - database.js — инициализация БД и NeDB-обёртка

2. МОДЕЛИ SEQUELIZE (3 модели):
   - User.js — модель пользователя
   - MenuItem.js — модель блюда меню
   - Order.js — модель заказа

3. NeDB-ИНТЕРФЕЙС: слой абстракции, транслирующий
   NeDB-стиль запросов в Sequelize → PostgreSQL

КОМПОНЕНТ БАЗЫ ДАННЫХ:
  PostgreSQL 16 с таблицами: users, menu_items, orders

СВЯЗИ МЕЖДУ КОМПОНЕНТАМИ:

Клиентская часть:
  HTML → CSS: подключение стилей
  HTML → JS: подключение скриптов
  JS  → localStorage: чтение/запись
  JS  → API: HTTP-запросы

Между клиентом и сервером:
  API (db.js) → Express.js (server.js): REST API (JSON)
  через HTTP-запросы (fetch)

Серверная часть:
  Express.js → NeDB API: вызов db.users/menu/orders
  NeDB API → database.js: трансляция запросов
  database.js → Модели Sequelize
  Модели → PostgreSQL: SQL-запросы

ОСОБЕННОСТИ:
- Чёткое разделение на клиентскую и серверную части
- NeDB-интерфейс изолирует логику приложения от СУБД
- Все модули имеют единую точку входа (app.js на клиенте,
  server.js на сервере)
- Внешняя зависимость: только Google Fonts (шрифты)
