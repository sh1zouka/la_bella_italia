============================================================
НАЗВАНИЕ ДИАГРАММЫ: Диаграмма активности оформления заказа
    (Activity Diagram)
============================================================

ОПИСАНИЕ:
Диаграмма активности моделирует поток управления процесса
оформления заказа в системе доставки «La Bella Italia».
Она показывает последовательность действий от момента
выбора блюд до получения заказа клиентом.

ОСНОВНЫЕ ЭТАПЫ ПРОЦЕССА:

1. ВЫБОР БЛЮД И КОРЗИНА
   - Пользователь просматривает меню
   - Добавляет блюда в корзину
   - Переходит на страницу корзины
   - Если корзина пуста — возврат к меню

2. ВВОД ДАННЫХ ДОСТАВКИ
   - Пользователь вводит ФИО, телефон и адрес
   - Клиентская валидация:
     * ФИО: два слова (кириллица/латиница, дефис)
     * Телефон: +7, 11 цифр
     * Адрес Москвы: тип улицы + номер дома
   - При ошибке — подсветка красным
   - При успехе — продолжение

3. ОТПРАВКА ЗАКАЗА
   - Опционально: использование бонусов
   - POST /api/orders на сервер
   - Проверка авторизации (401 → вход)

4. СЕРВЕРНАЯ ОБРАБОТКА
   - Расчёт:
     * total = Σ(price × quantity)
     * bonusDiscount = min(bonusPoints, floor(total × 0.3))
     * finalTotal = total - bonusDiscount
     * earnedBonuses = floor(finalTotal × 0.05)
   - Создание заказа в БД (INSERT INTO orders)
   - Обновление бонусов (UPDATE users)
   - Возврат номера заказа клиенту

5. СТРАНИЦА УСПЕХА
   - Отображение: номер заказа, итоговая сумма,
     начисленные бонусы
   - Очистка корзины

6. ЖИЗНЕННЫЙ ЦИКЛ ЗАКАЗА
   - Администратор меняет статус заказа
   - Цикл: confirmed → preparing → delivering → done
   - Или cancelled (отмена)
   - Пользователь отслеживает статус в профиле

7. ЗАВЕРШЕНИЕ
   - Заказ доставлен
   - Пользователь получает заказ

ОСОБЕННОСТИ:
- На каждом шаге предусмотрена обработка ошибок
- Валидация выполняется как на клиенте, так и на сервере
- Бонусная система интегрирована в процесс заказа
- Администратор управляет статусами вручную
- Пользователь может отслеживать заказ в реальном времени
