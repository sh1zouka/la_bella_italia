============================================================
НАЗВАНИЕ ДИАГРАММЫ: ER-диаграмма базы данных
    (Entity-Relationship Diagram)
============================================================

ОПИСАНИЕ:
ER-диаграмма отображает логическую структуру базы данных
системы доставки «La Bella Italia». База данных состоит из
4 таблиц, связанных между собой внешними ключами:

1. Таблица users (Пользователи)
   - Хранит данные о зарегистрированных пользователях
   - Ключевые поля: id (PK), email (UNIQUE), password (bcrypt),
     role (user/admin), bonusPoints
   - Связь: 1 пользователь → N заказов

2. Таблица orders (Заказы)
   - Хранит информацию о заказах
   - Содержит внешний ключ userId → users.id
   - Поля: total, bonusDiscount, finalTotal, earnedBonuses,
     status, deliveryData (JSONB)
   - Связь: 1 заказ → N позиций заказа

3. Таблица order_items (Позиции заказа)
   - Связующая таблица для нормализации данных
   - Обеспечивает связь M:N между заказами и блюдами меню
   - Содержит внешние ключи: orderId → orders.id,
     menuItemId → menu_items.id
   - Хранит quantity, а также цену и название на момент заказа
     (для сохранения исторических данных)

4. Таблица menu_items (Блюда меню)
   - Хранит 50 блюд в 5 категориях
   - Поля: category, name, price, weight, macros (JSONB),
     popular
   - NOTA: не имеет обратной связи к заказам напрямую —
     связь осуществляется через order_items

СВЯЗИ МЕЖДУ ТАБЛИЦАМИ (все таблицы связаны):
  users (1) ──── (N) orders (1) ──── (N) order_items (N) ──── (1) menu_items

  - users → orders: один ко многим (1:N)
  - orders → order_items: один ко многим (1:N)
  - order_items → menu_items: многие к одному (N:1)

ОСОБЕННОСТИ:
- В приложении данные заказа хранятся в JSONB-поле items,
  но для реляционной модели показана нормализованная
  структура через таблицу order_items
- JSONB-поля (macros, deliveryData, items) обеспечивают
  гибкость документной модели внутри реляционной БД
- Все таблицы имеют первичные ключи (id)
- Используются индексы для оптимизации запросов (по email,
  userId, status, category, popular)
