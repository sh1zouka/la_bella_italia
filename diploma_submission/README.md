**Квалификация:** Администратор баз данных  
**Стек:** Node.js + Express + PostgreSQL + Sequelize

## 🎯 Возможности

### Для клиента
- 🍕 Просмотр меню с фильтрацией по категориям (пицца, паста, закуски, десерты, напитки)
- 🔍 Поиск блюд по названию
- 🛒 Корзина с изменением количества и удалением
- 🎁 Бонусная программа: +100 при регистрации, 5% кэшбэк, скидка до 30%
- 📋 История заказов в личном кабинете
- ✅ Валидация полей доставки (имя, телефон +7, адрес Москвы)

### Для администратора
- 📦 Управление заказами (просмотр, смена статуса, удаление)
- 👥 Список пользователей
- 🍕 CRUD меню (добавление и удаление блюд)
- 📊 Статистика (количество заказов, выручка, средний чек)

## 🛠 Технологический стек

| Компонент | Технология |
|-----------|-----------|
| **Сервер** | Node.js + Express.js |
| **База данных** | PostgreSQL + Sequelize ORM |
| **Аутентификация** | express-session + bcryptjs |
| **Фронтенд** | HTML5 + CSS3 + Vanilla JavaScript |
| **Безопасность** | Helmet, express-rate-limit, httpOnly cookie, SameSite |
| **Тестирование** | Mocha + Chai + Supertest |
| **Валидация** | Клиентская + серверная |

### Особенности реализации

- **NeDB-совместимая обёртка**: код написан с использованием API NeDB (find, insert, update, remove), который через прослойку `database.js` перенаправляет запросы в PostgreSQL через Sequelize. Это позволяет легко мигрировать между СУБД без изменения логики приложения.
- **JSONB в PostgreSQL**: гибкие данные (состав заказа, макросы блюд) хранятся в JSONB-полях — баланс между документной и реляционной моделями.

```

    │
    ▼
REST API (Express.js)
    │
    ▼
NeDB-совместимый API слой (database.js)
    │
    ▼
Sequelize ORM
    │
    ▼
PostgreSQL
```

### Схема взаимодействия при заказе:

```
Клиент → Добавление в корзину (localStorage)
         → Заполнение формы доставки
         → POST /api/orders
              → Проверка аутентификации
              → Расчёт бонусов (5% кэшбэк, до 30% скидка)
              → Создание заказа в БД
              → Обновление бонусов пользователя
              → Ответ с номером заказа
```

## 💾 База данных

### Модели

#### User
| Поле | Тип | Описание |
|------|-----|----------|
| `_id` | SERIAL PK | Уникальный идентификатор |
| `name` | VARCHAR(255) | Имя пользователя |
| `email` | VARCHAR(255) UNIQUE | Email |
| `password` | VARCHAR(255) | Хеш пароля (bcrypt) |
| `phone` | VARCHAR(20) | Телефон |
| `role` | VARCHAR(20) | Роль: `user` или `admin` |
| `bonusPoints` | INTEGER | Бонусные баллы |
| `createdAt` | TIMESTAMP | Дата регистрации |

#### MenuItem
| Поле | Тип | Описание |
|------|-----|----------|
| `id` | SERIAL PK | Уникальный идентификатор |
| `category` | VARCHAR(50) | Категория: pizza, pasta, starters, desserts, drinks |
| `name` | VARCHAR(255) | Название блюда |
| `desc` | TEXT | Описание |
| `price` | INTEGER | Цена в рублях |
| `weight` | VARCHAR(20) | Вес порции |
| `img` | VARCHAR(100) | Emoji-иконка |
| `photo` | TEXT | URL фото |
| `macros` | JSONB | БЖУ: {kcal, protein, fat, carbs} |
| `popular` | BOOLEAN | Хит продаж |

#### Order
| Поле | Тип | Описание |
|------|-----|----------|
| `_id` | SERIAL PK | Уникальный идентификатор |
| `userId` | INTEGER FK → User._id | ID пользователя |
| `items` | JSONB | Состав заказа [{name, price, qty}] |
| `total` | INTEGER | Сумма заказа |
| `bonusDiscount` | INTEGER | Скидка бонусами |
| `finalTotal` | INTEGER | Итоговая сумма |
| `earnedBonuses` | INTEGER | Начислено бонусов |
| `deliveryData` | JSONB | {name, phone, address} |
| `status` | VARCHAR(50) | Статус: confirmed, preparing, delivering, delivered |
| `createdAt` | TIMESTAMP | Дата заказа |

## 🌐 REST API

| Метод | Эндпоинт | Доступ | Описание |
|-------|----------|--------|----------|
| POST | `/api/register` | Публичный | Регистрация пользователя |

## 🚀 Установка и запуск

### Требования
- Node.js 18+
- npm

### Быстрый запуск (NeDB — PostgreSQL не требуется)

```bash
# 1. Перейти в папку сервера
cd server

# 2. Установить зависимости
npm install

# 3. Запустить сервер (автоматически использует NeDB)
node server.js

# 4. Открыть в браузере:
#    http://localhost:3000
```

### Запуск с PostgreSQL (если требуется)

```bash
# 1. Убедитесь, что PostgreSQL запущен на localhost:5432
#    Создайте базу данных:
psql -U postgres -c "CREATE DATABASE italian_delivery;"

# 2. Перейти в папку сервера
cd server

# 3. Установить зависимости
npm install

# 4. Запустить с PostgreSQL
DB_ENGINE=postgres node server.js

# 5. Открыть в браузере:
#    http://localhost:3000
```

> **Важно:** По умолчанию сервер работает с NeDB (встроенная файловая БД) — не требует установки PostgreSQL. Для переключения на PostgreSQL установите переменную `DB_ENGINE=postgres`.

## 👥 Ролевая модель

| Роль | Права |
|------|-------|
| **Гость** (не авторизован) | Просмотр меню, фильтрация, поиск |
| **Клиент** (user) | Корзина, оформление заказа, история, бонусы |
| **Администратор** (admin) | Управление заказами, меню, пользователями, статистика |

### Бонусная система
- 🎁 **100 баллов** — при регистрации
- 💰 **5% кэшбэк** — с каждого заказа
- 🎯 **до 30% скидка** — можно оплатить бонусами
- 📊 **Уровни:** Новичок (0-499), Гурман (500-1499), Шеф (1500+)
